The Problem

Vision-only agents are slow & expensive

Most AI desktop agents screenshot → LLM → click. Repeat 10-20 times. There's a fundamentally better approach.

❌ Pure Vision Agents

  1. Screenshot the entire screen
  2. Send to GPT-4o / Claude ($$)
  3. Get back pixel coordinates
  4. Click at (x, y) — hope it's right
  5. Screenshot again... repeat 10x
~12s
Per task
15K
Tokens

✅ Clawd Cursor

  1. Parse task into subtasks (1 text call)
  2. Query UI Automation tree
  3. Find element by name — click directly
  4. No screenshots for 80% of tasks
  5. LLM vision only as fallback
~2s

Architecture

Parse → Accessibility → Action

A three-stage pipeline that tries the fastest approach first, falling back to vision only when needed.

1

Decompose Task

Single text-only LLM call breaks your request into atomic subtasks.

⚡ 200ms · 500 tokens
2

Action Router

Queries Windows UI Automation — same API as JAWS/NVDA screen readers. Finds elements by name, not coordinates.

✅ No LLM · 300ms · $0
3

Vision Fallback

Only when router can't handle it: resized screenshot → LLM → coordinates. Used for ~20% of tasks.

⚠️ Fallback · 2s
Features

Built for speed, safety, and reliability

🔍

UI Automation Tree

Queries the Windows accessibility API to find buttons, menus, and inputs by name — not pixel coordinates.

⚡

Zero-LLM Fast Path

Common tasks execute in milliseconds with zero AI calls. Open apps, type text, click elements — instantly.

🧠

Smart Decomposition

One cheap text-only LLM call breaks complex tasks into atomic steps the router can handle.

📸

Resized Screenshots

When vision is needed, screenshots are optimized to minimize token usage while preserving accuracy.

🛡️

Safety Tiers

Three-tier system: auto for safe actions, preview for typing, confirm for anything destructive.

🔌

VNC Protocol

Connects as a standard VNC client. Watch the AI work in real-time from your own viewer.

🟢 Auto

Open apps, navigate, read screen, switch windows. Executes immediately — no confirmation needed.

🟡 Preview

Type text, fill forms. Logged before executing so you can see exactly what's happening.

🔴 Confirm

Send messages, delete files, purchases. Pauses and waits for your explicit approval.

Get Started

Up and running in 2 minutes

terminal 
# Clone the repo
git clone https://github.com/AmrDab/clawd-cursor.git
cd clawd-cursor

# One-command setup (installs TightVNC + builds project)
powershell -ExecutionPolicy Bypass -File setup.ps1

# Add your AI API key
# Edit .env → AI_API_KEY=sk-...

# Start the agent
npm start -- --vnc-password yourpass

# Send a task
curl http://localhost:3847/task \
  -d '{"task": "Open Chrome and go to github.com"}'
The setup script handles: Node.js check · TightVNC download & install · npm install · TypeScript build · .env setup
Only need: Node.js 20+ · Windows · AI API key (Anthropic or OpenAI). Everything else is automated.
